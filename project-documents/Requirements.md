# Requirements.md — 環島中港通酒店套票監察系統

> **目標網址：** https://www.tilchinalink.com/promotions.php  
> **監察範圍：** 酒店套票（酒店住宿 + 車費 + 自助餐/早餐）  
> **技術棧：** GitHub Actions + Python + OpenRouter（免費模型）+ Discord Webhook  
> **版本：** 1.1 | 27 April 2026  
> **語言：** 繁體中文輸出

---

## 1. 背景及目標

環島中港通（Trans-Island Chinalink）是一間跨境客運公司，提供香港、澳門及中國大陸之間的客車及豪華轎車服務。其優惠頁面將酒店套票、演唱會、主題公園、車費折扣及充值獎賞等多種不同性質的優惠混合在同一列表中發布。

**業務需求：** 每日自動監察此頁面，篩選出符合條件的酒店套票（酒店住宿 + 車費 + 自助餐或早餐），以繁體中文格式發送至指定 Discord 頻道，時間為每日早上 09:00（HKT）。

### 2.2 優惠類別分析

經分析頁面內容（2026年4月27日），同一列表中存在以下優惠類別：

| 類別 | 實際例子 | 是否納入 |
|---|---|---|
| **A — 酒店套票**（住宿 + 車費 + 自助餐/早餐） | 酒店 + 交通 + 餐飲套票 | ✅ 納入 |
| B — 演唱會 / 表演 | 郭富城、黎明演唱會；澳門2049 | ❌ 排除 |
| C — 主題公園 / 景點 | 深圳華發雪 Bonski、長隆 | ❌ 排除 |
| D — 純車費折扣 | 買去程送回程（無住宿） | ❌ 排除 |
| E — 充值 / 現金獎賞 | WeChat Pay / 支付寶充值優惠 | ❌ 排除 |
| F — 新路線開通優惠 | 廣州西、福永、順德半價優惠 | ❌ 排除 |
| G — 售票中心折扣券 | 北角 $10 即減 | ❌ 排除 |

> ⚠️ **重要提示：** 部分優惠將酒店名稱列為**上落車地點**（例如：花山希爾頓歡朋酒店為新路線站點、佳和花園酒店為上車點）。這些屬於**純車費優惠**，必須排除 — 優惠中出現酒店名稱並不代表屬於酒店套票。

### 2.3 抓取注意事項

- 各優惠之間無語義化 HTML 容器（如 `<article>`、`<section>`）— 以純文字段落及 `...` 分隔
- 無 JSON-LD / 結構化 Schema 標記
- 已過期優惠仍保留在頁面上，並無任何「已過期」標示 — 日期驗證至關重要
- 個別優惠 URL 格式：`https://www.tilchinalink.com/promotions.php?id=XXX`
- 抓取時須從頁面鏈接中提取各優惠的 `id` 參數，以便在通知中提供直接連結

## 3. 範圍定義

### 3.1 合資格酒店套票 — 納入條件

只有同時符合以下**全部條件**的優惠，才會被視為合資格酒店套票：

| # | 條件 | 備注 |
|---|---|---|
| 1 | 包含酒店住宿（1晚或以上） | 必須為實際入住，而非僅於酒店大堂上落車 |
| 2 | 包含巴士／客車／豪華轎車交通（任何方向） | 可為單程或來回 |
| 3 | 包含自助餐**或**早餐（至少一項） | 必須包含餐飲項目 |
| 4 | 以套票形式（合併定價）出售 | 非單獨購買 |

### 3.2 自動排除條件

無論優惠標題或內容如何提及酒店，只要優惠主要屬於以下類型，均予排除：

- 演唱會、表演或音樂節套票
- 主題公園、雪場或遊樂場套票
- 觀光遊覽或導遊套票
- 純車費折扣（酒店僅為上落車地點）
- 充值積分、現金回贈或優惠券
- 新路線開通優惠（酒店僅為登車站點）

### 3.3 過期處理

- 將每個優惠的有效期與**執行日期**（已注入提示詞）進行比對
- 已過期優惠須靜默略過，不得出現在輸出內容中
- 沒有列明結束日期的優惠，視為**仍然有效**

## 4. 功能需求

### FR-1 — 網頁抓取

| ID | 需求 |
|---|---|
| FR-1.1 | 以 HTTP GET 方式抓取 `https://www.tilchinalink.com/promotions.php`（繁體中文版本，不加 `?lang=en`） |
| FR-1.2 | 同時抓取第 2、3 頁（`?page=2`、`?page=3`）以獲取較舊的有效優惠 |
| FR-1.3 | 若整頁優惠均早於 6 個月前發布，則停止翻頁 |
| FR-1.4 | 在 HTTP 請求標頭中設置有效的瀏覽器 `User-Agent` |
| FR-1.5 | 設置 30 秒請求超時；遇 `ConnectionError` 或 HTTP 5xx 時重試一次 |
| FR-1.6 | 移除 `<nav>`、`<footer>`、`<script>`、`<style>`、`<img>` 標籤後再提取文字 |
| FR-1.7 | **從頁面連結中提取個別優惠的 URL**（格式：`?id=XXX`），並與對應優惠文字配對 |
| FR-1.8 | 如主要內容選擇器失效，則回退至完整 `<body>` 文字 |

### FR-2 — 變更偵測

| ID | 需求 |
|---|---|
| FR-2.1 | 計算所有已抓取頁面合併原始文字的 SHA-256 雜湊值 |
| FR-2.2 | 將目前雜湊值與儲存於倉庫的 `last_hash.txt` 進行比對 |
| FR-2.3 | 若雜湊值**相同**：發送簡短「無變更」Discord 通知並退出 — **不調用 LLM** |
| FR-2.4 | 若雜湊值**不同**：繼續進行 LLM 摘要，並在成功後更新 `last_hash.txt` |
| FR-2.5 | 每次執行後將更新的 `last_hash.txt` 提交回倉庫 |

### FR-3 — LLM 分類及摘要

| ID | 需求 |
|---|---|
| FR-3.1 | 使用推薦模型（見第 6 節）調用 OpenRouter API |
| FR-3.2 | 在系統提示詞中注入 `TODAY_DATE`，讓模型能驗證優惠有效期 |
| FR-3.3 | 提示詞指示模型分類並篩選 — 僅輸出符合第 3 節條件的酒店套票 |
| FR-3.4 | 提示詞明確列出排除類別（演唱會、主題公園、純車費優惠等） |
| FR-3.5 | 輸出格式包含：套票名稱、有效期、價格、餐飲詳情、住宿詳情、訂票渠道、備注及優惠 URL |
| FR-3.6 | **輸出格式不包含交通時間表及路線詳情**（用戶可自行查閱官方網站） |
| FR-3.7 | 輸出結尾加入統計行：`✅ 找到 N 個酒店套票 | 🚫 已排除 M 個非酒店優惠` |
| FR-3.8 | 若無合資格套票：輸出 `🔍 今日無酒店套票。已排除 M 個其他優惠。` |
| FR-3.9 | **所有輸出使用繁體中文** |
| FR-3.10 | 傳送至 LLM 的內容上限為 12,000 tokens |
| FR-3.11 | 實施模型備用鏈：主要 → 次要 → 第三選擇（見第 6 節） |

### FR-4 — Discord 通知

| ID | 需求 |
|---|---|
| FR-4.1 | 透過 Incoming Webhook URL 向 Discord 頻道發送摘要 |
| FR-4.2 | 根據結果類型使用不同訊息標題（找到套票 / 無套票 / 無變更 / 錯誤） |
| FR-4.3 | 每個套票項目須包含**官方網站的個別優惠直接連結** |
| FR-4.4 | 每條訊息頁腳包含完整優惠頁面連結 |
| FR-4.5 | 抓取失敗時：發送含失敗原因的錯誤警報 |
| FR-4.6 | LLM 失敗時：發送錯誤警報（不發送空白或不完整的輸出） |
| FR-4.7 | Discord 訊息總長度不得超過 2,000 字元（Discord 限制） |

### FR-5 — 排程及觸發

| ID | 需求 |
|---|---|
| FR-5.1 | 透過 GitHub Actions cron 每日自動執行：**01:00 UTC（09:00 HKT）** |
| FR-5.2 | 支援 `workflow_dispatch` 手動觸發，用於即時執行及測試 |
| FR-5.3 | 工作流程必須具備 `contents: write` 權限以提交 `last_hash.txt` |

## 5. 非功能需求

| 屬性 | 需求 |
|---|---|
| **費用** | $0 — GitHub Actions 免費版（每月 2,000 分鐘）；OpenRouter 免費模型 |
| **執行時間** | 完整流程在 3 分鐘內完成 |

| **可維護性** | 提示詞、模型 ID、URL 及篩選條件以常數形式定義；無硬編碼業務邏輯 |

## 6. OpenRouter 免費模型推薦

> ⚠️ **香港帳戶限制：** 由於 OpenRouter.ai 帳戶位於香港，以下提供商的模型**不可使用**：
> - ❌ **Anthropic**（Claude 系列）
> - ❌ **OpenAI**（GPT 系列）
> - ❌ **Google**（Gemini / Gemma 系列）
>
> 以下推薦均為非上述提供商的免費模型。

### 6.1 模型優先級列表

| 優先級 | 模型 ID | 提供商 | 上下文 | 繁體中文 | 狀態 |
|---|---|---|---|---|---|
| **1 — 主要** | `qwen/qwen3-next-80b:free` | Alibaba（阿里巴巴） | 262K | ⭐⭐⭐⭐⭐ 原生 | 穩定免費 |
| **2 — 備用** | `z-ai/glm-4.5-air:free` | Z.AI（智譜 AI） | 131K | ⭐⭐⭐⭐⭐ 原生 | 穩定免費 |
| **3 — 備用** | `meta-llama/llama-3.3-70b-instruct:free` | Meta | 131K | ⭐⭐⭐ 良好 | 穩定免費 |

### 6.2 選擇理由

**主要：`qwen/qwen3-next-80b:free`（Qwen3-Next 80B）**
- 阿里巴巴原生中文大模型，對繁體中文（粵港澳用語）理解能力最佳
- 針對 RAG 及 Agentic 工作流程優化，分類準確性高
- 262K token 上下文 — 單次 API 調用即可處理所有分頁優惠內容
- 完全免費，非 Google / OpenAI / Anthropic

**次要：`z-ai/glm-4.5-air:free`（GLM-4.5-Air）**
- 智譜 AI 原生中文模型，繁體中文輸出質量優秀
- 混合推理模式（思考模式 / 非思考模式），可提升日期驗證準確性
- 131K token 上下文，MoE 架構效率高

**第三選擇：`meta-llama/llama-3.3-70b-instruct:free`（Llama 3.3 70B）**
- GPT-4 同等整體性能
- 中文理解能力遜於 Qwen / GLM，但作為最後備用方案仍可接受
- 131K token 上下文

> ⚠️ **DeepSeek 注意事項：** 雖然 DeepSeek 模型中文能力優秀且免費，但其伺服器位於中國大陸，所有提示詞均會用於模型訓練且無法退出，存在數據私隱風險。本方案不建議使用。

### 6.3 頻率限制說明

OpenRouter 免費模型限制：每分鐘 20 個請求，每日 200 個請求。
本流程每日最多使用 **0–1 次 LLM 調用**（頁面無變更時為 0 次），遠低於限制。

## 7. Discord 訊息格式（繁體中文）

### 找到酒店套票時：
```
🏨 環島中港通 酒店套票快訊 | 27/04/2026

🏨 **【套票名稱】**
📅 有效期：【日期】
💰 價格：【每位價格或「請查閱官網」】
🍽️ 餐飲：【自助餐 / 早餐詳情】
🛏️ 住宿：【晚數 + 房型】
📲 訂票：【渠道：微信小程式 / 支付寶 / 售票中心】
📝 備注：【主要限制或亮點，最多一句】
🔗 優惠詳情：https://www.tilchinalink.com/promotions.php?id=XXX

✅ 找到 1 個酒店套票 | 🚫 已排除 6 個非酒店優惠
━━━━━━━━━━━━━━━━━━━━━━
🔗 查閱所有優惠 → https://www.tilchinalink.com/promotions.php
```

### 無酒店套票時：
```
🔍 環島中港通 酒店套票 | 27/04/2026
🔍 今日無酒店套票。已排除 7 個其他優惠。
🔗 https://www.tilchinalink.com/promotions.php
```

### 頁面無變更時：
```
ℹ️ 環島中港通 酒店套票 — 今日頁面無更新（27/04/2026）
```

### 發生錯誤時：
```
⚠️ 優惠監察機器人錯誤 | 27/04/2026
【抓取失敗 / LLM 失敗】：【錯誤信息】— 請手動檢查。
```
